# CLAUDE.md - Coding

## Para qué

Desarrollo, code review, debugging, refactoring.

---

## Salida

- Código primero. Explicación después, solo si no es obvio.
- Sin prosa innecesaria.
- Sin boilerplate a menos que se pida.

## Reglas de código

- Solución más simple que funcione.
- Sin abstracciones prematuras.
- Sin características especulativas.
- Leer el archivo antes de modificarlo.
- Sin docstrings en código no modificado.
- Tres líneas similares son mejor que una abstracción prematura.

## Code review

- Decir el bug. Mostrar la fix. Parar.
- Sin sugerencias fuera del alcance.
- Sin cumplidos.

## Debugging

- No especular sin leer el código primero.
- Decir qué encontré, dónde, y la fix.
- Si no está claro: decir que no se sabe. No guess.

## Formato

- Sin em dashes, smart quotes o Unicode decorativo.
- Guiones normales y comillas rectas.
